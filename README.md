## :eye: About

<br>

<div align="center">

**`AirBnB clone - the complete project`** is a basic and comprehensive clone of the `AirBnB` platform, developed in **four progressive parts**.
<br>
<br>
It combines `backend` and `frontend` technologies to simulate core functionalities of the `AirBnB` website.
<br>
The project is built using `Python`, `Flask`, `HTML`, `CSS`, `JavaScript`, and `MySQL` with `SQL`, with a focus on **dynamic user interactions**, **responsive design** and efficient **data management**.
<br>
The goal is to create a fully functional web application that serves as an interface between users and backend data models, facilitating dynamic content rendering and efficient `HTTP` request handling.
<br>
<br>
This project has been created by **[Holberton School](https://www.holbertonschool.com/about-holberton)** to enable every student to understand how to build a fully functional web application.
<br>
<br>
Through hands-on development of core components, we gain practical experience in backend systems, data modeling, software architecture, and front-end web development skills.

</div>

<br>
<br>

## :hammer_and_wrench: Tasks

<br>

- **`Console Creation`**:

  - Develop a command-line interface (`CLI`) to manage the application's data models
  
  - Tasks include creating, updating, and deleting objects, as well as implementing a storage system.

<br>

- **`Database Management with MySQL`**:

  - Set up `MySQL` databases for development and testing environments using `SQL` scripts
  
  - Tasks include creating databases and users, granting privileges, and managing data through `SQL` queries
  
  - Integrate `MySQL` with the application for data persistence and retrieval, ensuring efficient data management and query optimization.

<br>

- **`Web Static`**: 

  - Develop the static front-end using `HTML` and `CSS`
  
  - Tasks include inline styling, head styling, and responsive design.

<br>

- **`Web Framework`**: 

  - Implement a web application using `Flask` to handle `HTTP` requests and dynamic content rendering
  
  - Tasks include setting up `Flask` routes, creating templates, and managing data flow.

<br>

- **`Web Dynamic`**: 

  - Enhance the application with `JavaScript` for dynamic interactions without full page reloads
  
  - Tasks include making `API` requests, `DOM` manipulation, and event handling with `jQuery`.

<br>

- **`Final Integration`**: 

  - Combine all components to create a **fully functional web application**
  
  - Tasks include integrating `frontend` and `backend`, optimizing performance, and ensuring **cross-browser compatibility**.

## :package: Installation

<br>

- Clone this repository: `git clone "https://github.com/RazikaBengana/holbertonschool-AirBnB_clone_v4.git"`

- Access the project directory

- Run `hbnb` (interactively): `./console` and enter command

- Run `hbnb` (non-interactively): `echo "<command>" | ./console.py`

## :keyboard: Basic Usage

<br>

<details>
<summary> :white_check_mark: <h3><a href="console.py"><code>console.py</code></a></h3> </summary>

<br>
<br>

The console contains the entry point of the command interpreter.

<br>

- Functionalities available:

  - Create a new object (ex: a new `User` or a new `Place`)
  
  - Retrieve an object from a file, a database, etc... 
  
  - Do operations on objects (count, compute stats, etc...)
  
  - Update attributes of an object 
  
  - Destroy an object

<br>
<br>

| Command                  | Description                                                                                                                                 |
|--------------------------|---------------------------------------------------------------------------------------------------------------------------------------------|
| `EOF`                    | Exits console.                                                                                                                              |
| `quit`                   | Exits console.                                                                                                                              |
| `<emptyline>`            | Overwrites default emptyline method and does nothing.                                                                                       |
| `create`                 | Creates a new instance of `BaseModel`, saves it (to the JSON file), and prints the id.                                                      |
| `destroy`                | Deletes an instance based on the class name and id (saves the change into the JSON file).                                                   |
| `show`                   | Prints the string representation of an instance based on the class name and id.                                                             |
| `all`                    | Prints all string representation of all instances based or not on the class name.                                                           |
| `update`                 | Updates an instance based on the class name and id by adding or updating attribute (save the change into the JSON file).                    |

</details>

<br>

The `BaseModel` class from which future classes will be derived.

<br>

| Method                            | Description                                                                                                                        |
|-----------------------------------|------------------------------------------------------------------------------------------------------------------------------------|

| `__str__(self)`                   | String representation of the `BaseModel` class.                                                                                    |
| `save(self)`                      | Updates the attribute `updated_at` with the current datetime.                                                                      |
| `to_dict(self)`                   | Returns a dictionary containing all keys/values of the instance.                                                                   |
| `delete(self)`                    | Removes the current instance from storage by calling the `delete` method of storage.                                                 |

<br>

| Model                    | Description                                                                                                                        |

<details>
<summary> :white_check_mark: <h3><a href="models/engine"><code>models/engine</code></a></h3> </summary>

<br>

- #### [engine/file_storage.py](models/engine/file_storage.py)

<br>

The `FileStorage` class that handles JSON serialization and deserialization.

<br>

| Method                   | Description                                                          |
|--------------------------|----------------------------------------------------------------------|
| `all(self, cls=None)`    | Returns the dictionary `__objects`.                                  |
| `new(self, obj)`         | Sets in `__objects` the `obj` with key `<obj class name>.id`.        |
| `get(self, cls, id)`     | Retrieves an object by its `class` and `ID`.                         |
| `count(self, cls=None)`  | Returns count of objects in storage, optionally filtered by `class`. |
| `save(self)`             | Serializes `__objects` to the JSON file (path: `__file_path`).       |
| `reload(self)`           | Deserializes the JSON file to `__objects`.                           |
| `delete(self, obj=None)` | Deletes obj from `__objects` if it exists.                           |
| `close(self)`            | Calls `reload()` method for deserializing the JSON file to objects.  |

<br>
<br>

- #### [engine/db_storage.py](models/engine/db_storage.py)

<br>

The `DBStorage` class that handles all database operations for the models using `SQLAlchemy`.

<br>

| Method                   | Description                                                                                         |
|--------------------------|-----------------------------------------------------------------------------------------------------|
| `__init__(self)`              | Initializes a new `DBStorage` instance, setting up database connection using environment variables. |
| `all(self, cls=None)`              | Queries the current database session and returns a dictionary of objects.                           |
| `new(self, obj)`         | Adds a new object to the current database session.                                                  |
| `get(self, cls, id)`             | Retrieves an object by its class and id from storage.                                               |
| `count(self, cls=None)`           | Returns the number of objects in storage, optionally filtered by class.                             |
| `save(self)`             | Commits all changes in the current session to the database.                                         |
| `delete(self, obj=None)`           | Deletes an object from the current session if it exists.                                            
| `reload(self)`              | Creates all tables in the database and initializes a new session.                                   |
| `close(self)`         | Closes the current session, freeing resources.                                                      |

</details>

<br>

A Fabric script that:

- Stores archives in a `versions` directory

- Returns the archive path if successful, `None` if failed

- Handles cleanup of temporary files

- Returns `True` if successful, `False` if failed
